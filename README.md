# Домашнее задание к занятию "`Репликация и масштабирование. Часть 1`" - `Рунов ВВ`


### Задание 1
  На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия.

Ответить в свободной форме.

---
Master-Slave
Архитектура: более простая, для систем с высокой нагрузкой на чтение.
Мастер доступен на чтение и запись операций, а один или несколько Slave с него реплицируют и ддоступны только на чтение.

Синхронизация:
   – Слейвы периодически обновляют свои данные, синхронизируясь с мастером (асинхронно или синхронно).
Отказоустойчивость:
   – Если мастер выходит из строя, один из slave нужно вручную переключить слейв на роль мастера.
Применение:
   – Рекомендуется там, где требуется доступность данных на чтение и минимальная запись.

Master-Master

Архитектура: более сложная,
   – Два или более Мастер-серверов могут принимать записи и синхронизировать данные друг с другом.
   – дает разделение нагрузки на запись., тк возможна на любом из серверов. 

Синхронизация:
   – Необходимо обеспечить механизмы для разрешения конфликтов при записи, так как оба узла могут изменять одни и те же данные.

Отказоустойчивость:
   – Если один из серверов выходит из строя, другой  продолжает работать.

 Применение:
   – Подходит для распределенных систем.
